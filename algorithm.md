## 1. 공통 개념

### 시간감쇠계수 (TD, Time Decay)

오래된 콘텐츠일수록 점수를 낮춘다.

```
TD = 1 / (경과시간(시간) + 2)^0.5
```

| 경과시간 | TD 값 |
|---------|-------|
| 방금 (0h) | 0.707 |
| 1시간 | 0.577 |
| 6시간 | 0.354 |
| 24시간 (1일) | 0.196 |
| 168시간 (7일) | 0.077 |
| 720시간 (30일) | 0.037 |


### 행동가중치 (AW, Action Weight)

유저가 취한 행동의 강도. 값이 클수록 강한 관심 표현.

| 행동 | 가중치 |
|------|--------|
| VIEW | 1 |
| LIKE | 3 |
| COMMENT | 4 |
| SCRAP | 5 |
| FOLLOW | 6 |
| POST | 7 |
| JOIN | 8 |


### 반응가중치 (RW, Reaction Weight)

콘텐츠가 받은 반응의 강도. 품질 점수 계산에 사용.

| 반응 | 가중치 |
|------|--------|
| VIEW | 0.2 |
| LIKE | 2 |
| COMMENT | 4 |
| SCRAP | 5 |
| FOLLOW | 6 |
| JOIN | 8 |
| CTR | 10 |

## 2. 데이터 수집 구조

### TBL_USER_ACTION_LOG (KNOWRA_USER)

유저가 취한 모든 행동을 시계열로 기록. 삭제하지 않는 append-only 로그.

| 컬럼 | 설명 |
|------|------|
| USER_SN | 행동 주체 |
| TARGET_TYPE | COMM / COMM_POST / POST / USER |
| TARGET_SN | 대상 SN |
| ACTION_TYPE | VIEW / LIKE / COMMENT / SCRAP / POST / JOIN / FOLLOW |
| REG_DT | 행동 발생 일시 |

**기록 시점**

| 행동 | 기록 조건 |
|------|-----------|
| VIEW | 게시글 조회 시 (토큰 있는 경우만) |
| LIKE | 좋아요 신규 등록 시 |
| COMMENT | 댓글 작성 시 |
| SCRAP | 게시글 저장 시 |
| POST | 게시글 신규 작성 시 |
| JOIN | 커뮤니티 가입 ACTIVE 확정 시 |
| FOLLOW | 팔로우 활성화 시 |


### TBL_USER_INTEREST_SCORE (KNOWRA_USER)

유저의 대상별 관심도를 단일 점수로 유지하는 현재 상태 스냅샷.
로그와 달리 가역적(취소 시 점수 차감).

| 컬럼 | 설명 |
|------|------|
| USER_SN | 유저 SN |
| TARGET_TYPE | COMM / COMM_POST / POST / USER / TAG |
| TARGET_SN | 대상 SN |
| SCORE | 누적 관심도 (0 이상, 음수 불가) |

**점수 변동 규칙**

| 행동 | TARGET_TYPE | 변동 |
|------|-------------|------|
| 커뮤니티 JOIN | COMM | +8 |
| 커뮤니티 LEAVE | COMM | -8 |
| 커뮤니티 게시글 LIKE(UP) | COMM_POST | +3 |
| 커뮤니티 게시글 LIKE 취소 | COMM_POST | -3 |
| 일반 게시글 LIKE(UP) | POST | +3 |
| 일반 게시글 LIKE 취소 | POST | -3 |
| FOLLOW | USER | +6 |
| UNFOLLOW | USER | -6 |
| 게시글 LIKE(UP) | TAG (게시글 태그 각각) | +3 |
| 게시글 LIKE 취소 | TAG | -3 |
| 게시글 COMMENT | TAG | +4 |
| 게시글 SCRAP | TAG | +5 |

> 점수는 `Math.max(0, score + delta)`로 0 미만으로 내려가지 않는다.
> 비동기(@Async) + 독립 트랜잭션(REQUIRES_NEW)으로 메인 요청에 영향을 주지 않는다.

## 3. 유저 관심사 프로필

유저의 관심 패턴을 세 가지 차원으로 수치화한다.

```
유저태그점수(tag_sn)       = Σ(행동가중치 × 시간감쇠계수)   -- 특정 태그와 관련된 모든 행동
유저커뮤니티점수(comm_sn)  = Σ(행동가중치 × 시간감쇠계수)   -- 특정 커뮤니티와 관련된 모든 행동
유저작성자점수(user_sn)    = Σ(행동가중치 × 시간감쇠계수)   -- 특정 작성자와 관련된 모든 행동
```

> 현재 구현: LIKE/COMMENT/SCRAP 발생 시 해당 게시글의 태그들을 조회해 TBL_USER_INTEREST_SCORE (TARGET_TYPE='TAG')에 비동기 누적.
> 시간감쇠 적용은 실시간 집계 단계(미구현)에서 처리 예정.

## 4. 커뮤니티 탐색 알고리즘

### 4-1. 점수 구성 요소

#### 커뮤니티행동유사점수
해당 커뮤니티에서 유저가 수행한 행동 누적값.

```
커뮤니티행동유사점수 =
  (VIEW 수 × 1) + (LIKE 수 × 3) + (COMMENT 수 × 4) + (SCRAP 수 × 5) + (POST 수 × 7)
```
> 현재 구현: TBL_USER_INTEREST_SCORE (TARGET_TYPE=COMM) 값 사용 (JOIN/LEAVE 기반 proxy)


#### 커뮤니티관계점수
유저와 커뮤니티의 관계 강도.

```
커뮤니티관계점수 =
  (가입 여부 × 20) + (최근 방문 여부 × 10) + (유저커뮤니티점수 × 1.5)
```
> 현재 구현: 가입 여부 × 20 + 최근7일 방문 여부 × 10 + interestScore × 1.5


#### 커뮤니티품질점수
최근 7일간의 활동 지표로 커뮤니티의 건강도를 측정.

```
커뮤니티품질점수 =
  (최근7일 게시글 수 × 3) + (최근7일 댓글 수 × 2) + (최근7일 조회수 × 0.2)
  + (최근7일 활동 유저 수 × 4) + (최근7일 신규 팔로우 수 × 5)
```
> 현재 구현: 게시글 수 × 3 + 댓글 수 × 2 + 조회수 × 0.2 + 활동유저 수 × 4 + 신규가입자 수 × 5
> (TBL_COMM_POST 7일 집계 + TBL_COMM_MBR 신규 ACTIVE 집계)


#### 커뮤니티신선도점수
마지막 게시글 기준 시간감쇠 적용.

```
커뮤니티신선도점수 = 100 × TD(최근 게시글 경과시간)
```


#### 커뮤니티탐험점수
유저에게 아직 익숙하지 않은 커뮤니티를 발굴할수록 높아짐.

```
익숙도점수 = min(1.0, interestScore / 8.0)   -- 8 = JOIN 1회 기준
커뮤니티탐험점수 = (1 - 익숙도점수) × 20
```


#### 커뮤니티개인화총점

```
총점 = 행동유사 × C2 + 관계 × C3 + 품질 × C4 + 신선도 × C5 + 탐험 × C6
```

| 가중치 | 값 | 설명 |
|--------|----|------|
| C2 | 0.25 | 커뮤니티행동유사점수 |
| C3 | 0.35 | 커뮤니티관계점수 |
| C4 | 0.25 | 커뮤니티품질점수 |
| C5 | 0.10 | 커뮤니티신선도점수 |
| C6 | 0.05 | 커뮤니티탐험점수 |

> C1(관심사일치): 유저 TAG 관심도 점수 × 커뮤니티 태그 출현 횟수로 계산.


### 4-2. 섹션별 최종 점수

| 섹션 | 공식 | 특징 |
|------|------|------|
| **맞춤 (curated)** | `총점 × 1.0` | 미가입 커뮤니티만, 상위 10개 |
| **인기 (popular)** | `총점 × 0.7 + 품질 × 0.3` | 전체 대상, 상위 10개 |
| **일반 (filter)** | `총점 × 0.5 + 탐험 × 0.5` | 전체 대상, offset 20개, ctgrSn/keyword 필터 지원 |


### 4-3. 후보 커뮤니티 수집 조건

```
actvtnYn = 'Y'
AND stat = 'Y'
AND (prvcyStng IN ('public', 'anonymous') OR 유저가 ACTIVE 멤버)
```


### 4-4. API

| 엔드포인트 | 인증 | 설명 |
|------------|------|------|
| `GET /api/community/explore` | 필수 | 맞춤 10 + 인기 10 |
| `GET /api/community/explore/filter` | 선택 | 일반 필터, pageIndex / ctgrSn / keyword |

## 5. 게시글 개인화 알고리즘

> 미구현 — 설계 기준

### 5-1. 점수 구성 요소

#### 게시글관심사일치점수
```
Σ(유저태그점수(tag) × 게시글태그가중치)
```

#### 게시글행동유사점수
```
(유사 게시글 VIEW 수 × 1) + (LIKE 수 × 3) + (COMMENT 수 × 4) + (SCRAP 수 × 5)
```

#### 게시글관계점수
```
(유저커뮤니티점수 × 1.2) + (유저작성자점수 × 1.0)
+ (팔로우한 커뮤니티 여부 × 20) + (팔로우한 작성자 여부 × 15)
```

#### 게시글품질점수
```
(LIKE 수 × 2) + (COMMENT 수 × 4) + (SCRAP 수 × 5) + (VIEW 수 × 0.2) + (CTR × 10)
```

#### 게시글신선도점수
```
100 × TD(게시글 작성 경과시간)
```

#### 게시글탐험점수
```
(1 - 익숙도점수) × 새로움점수 × 20
```

#### 게시글개인화총점
```
총점 = 관심사일치 × W1 + 행동유사 × W2 + 관계 × W3 + 품질 × W4 + 신선도 × W5 + 탐험 × W6
```

## 6. 피드 알고리즘

> 미구현 — 설계 기준

| 피드 | 공식 |
|------|------|
| **전체 피드** | `게시글개인화총점 × 1.0` |
| **팔로잉 피드** | `총점 × 0.7 + 팔로우커뮤니티 × 20 + 팔로우작성자 × 15` |
| **인기 피드** | `총점 × 0.6 + 품질점수 × 0.4` |
| **최신 피드** | `총점 × 0.7 + 신선도점수 × 0.3` |

## 7. 핫글 슬라이더

> 미구현 — 설계 기준

최근 1시간 반응속도 기반으로 실시간 급상승 게시글을 선별.

```
반응속도점수 =
  (최근1시간 LIKE 수 × 2) + (COMMENT 수 × 4) + (SCRAP 수 × 5) + (VIEW 수 × 0.2)

핫글슬라이더점수 =
  (게시글개인화총점 × 0.5) + (품질점수 × 0.2) + (신선도점수 × 0.1) + (반응속도점수 × 0.2)
```

## 8. 구현 현황

| 기능 | 상태 | 비고 |
|------|------|------|
| TBL_USER_ACTION_LOG 기록 | ✅ 완료 | VIEW(게시글/커뮤니티), LIKE, COMMENT, SCRAP, POST, JOIN, FOLLOW |
| TBL_USER_INTEREST_SCORE 업데이트 | ✅ 완료 | JOIN/LEAVE(COMM), LIKE/UNLIKE(POST/COMM_POST), FOLLOW/UNFOLLOW(USER), LIKE/COMMENT/SCRAP(TAG) |
| 커뮤니티 탐색 (맞춤/인기/일반) | ✅ 완료 | `/api/community/explore`, `/api/community/explore/filter` |
| 커뮤니티 탐색 C1 (관심사일치) | ✅ 완료 | 유저 TAG 관심도 × 커뮤니티 태그 출현 횟수 집계 |
| 커뮤니티 관계점수 최근방문 | ✅ 완료 | getCommunity() VIEW 로그 → 7일 내 방문 여부 탐색 점수 +10 반영 |
| 커뮤니티 품질점수 활동유저/신규팔로우 | ✅ 완료 | 활동유저 countDistinct + TBL_COMM_MBR 신규가입자 집계 |
| 게시글 개인화 피드 | ✅ 완료 | `/api/feed/personalized`, `/api/feed/following`, `/api/feed/popular`, `/api/feed/latest` |
| 핫글 슬라이더 | ⏳ 미구현 | 1시간 윈도우 집계 필요 |
